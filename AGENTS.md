# Инструкции для AI-агентов

Этот файл содержит инструкции и контекст для AI-агентов (OpenCode и подобных), работающих с проектом Memory Game.

## Контекст проекта

**Memory Game** — браузерная игра на совпадение пар карточек с таймером и счетчиком ходов.

**Основные технологии:**
- HTML5, CSS3, Vanilla JavaScript
- Локальное хранилище (localStorage) для лучших результатов
- Модульная архитектура с классами

## Ожидания от диалога

Учти ожидания пользователя от диалога с LLM-агентом:

- Инфостиль из книги "Пиши, сокращай"
- Короткие тезисные и лаконичные сообщения
- Без "обсасывания" банальностей
- В форматировании текста вместо таблиц использовать обычные заголовки и абзацы текста
- Общение на русском языке, но с широким использованием технических терминов на английском

## Ключевые компоненты

### Классы
- `Card` — класс карточки с методами открытия/закрытия
- `Timer` — таймер игры с паузой/возобновлением
- `ScoreManager` — управление результатами и localStorage

### Основные функции
- `createCards()` — создает пары карточек
- `shuffleCards()` — перемешивает карточки
- `handleCardClick()` — обрабатывает клики по карточкам
- `checkMatch()` — проверяет совпадения пар
- `saveBestScore()` — сохраняет лучший результат

## Стиль кода

### JavaScript
- Использовать современный ES6+ синтаксис
- Комментарии на русском языке для сложной логики
- Именование переменных в camelCase
- Классы с заглавной буквы
- Константы в UPPER_SNAKE_CASE

### CSS
- Использовать CSS Custom Properties для цветов
- БЭМ-подобная методология для именования классов
- Мобильная верстка с медиа-запросами

## Конфигурация OpenCode

Пример конфига в `.opencode/config.yaml`:
```yaml
model: deepseek/deepseek-chat
temperature: 0.7
max_tokens: 4000
context_window: 16000
system_prompt: |
  Ты помогаешь разрабатывать игру Memory Game.
  Проект использует Vanilla JavaScript, HTML5 и CSS3.
  Следуй инструкциям из AGENTS.md.
  Будь лаконичным и конкретным в ответах.
```

## Частые задачи

### Добавление новой функциональности
1. Проверить спецификацию в `spec.md`
2. Создать новую ветку `feature/название-фичи`
3. Реализовать изменения
4. Протестировать в разных браузерах
5. Создать pull request

### Исправление багов
1. Воспроизвести проблему
2. Найти root cause в коде
3. Исправить с минимальными изменениями
4. Добавить тест для предотвращения регрессии

### Рефакторинг
1. Сохранить существующую функциональность
2. Улучшить читаемость кода
3. Удалить дублирование
4. Обновить документацию при необходимости
